# Security Policy

## Scope

This repository contains compliance templates, guides, and a client-side HTML tool. There is no server-side code, API, or data storage.

The interactive [Compliance Toolkit](tools/compliance-toolkit.html) runs entirely in the user's browser. No data is transmitted, stored, or processed externally.

## Reporting a Vulnerability

If you discover a security issue in this repository (e.g., a cross-site scripting vulnerability in the HTML tool, or a supply-chain concern with linked resources), please report it responsibly:

1. **Do not open a public issue.**
2. Email **dev@dishine.it** with the subject line `[SECURITY] ai-compliance-framework`.
3. Include a description of the vulnerability, steps to reproduce, and any suggested fix.
4. We will acknowledge receipt within 48 hours and provide a resolution timeline.

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 2.0.x   | ✅ Yes    |
| 1.x.x   | ❌ No     |

## Disclaimer

This repository does not process, store, or transmit personal data. The templates and tools are informational resources, not software services. The security policy applies to the repository content and the client-side HTML toolkit only.
